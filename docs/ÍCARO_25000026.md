# 1. Definição do MVP


 O meu MVP contempla o processo de vendas e o controle de estoque com forme o que foi vendido, notificando os estatuas da compra, notificando quanto o produto está próximo de zerar e não permitindo vender produtos que não tem.
 O MVP não contempla a parte da verificação da receitas para venda.
 Essas decisões foram tomadas para evitar a fraude de receitas médicas, pois é mais fácil para uma pessoa identificar quando é uma receita falsa do que uma máquina, além de manter a privacidade dos clientes.

# 2. Regras de Negócio (mínimo: 5)
Liste e descreva **cada RN** de forma clara.

**RN01 —**  Os produtos que estão zerados no estoque ** Não podem ser vendidos **.
**RN02 —**  Toda vez que uma compra é realizada há uma **atualização automática no estoque dos produtos vendidos**, assim evitando divergências. 
**RN03 —**  Sempre que o estoque de um produto chegar em um valor pré determinado ele irá **emitir um aviso de estoque baixo** para ser reposto.
**RN04 —**  Sempre que uma venda é realizada ela entra no estatuas de **"Aberta"** enquanto não for paga, só alterando o estatuas quando o pagamento é recebido.
**RN05 —**  No casos de grande demora para receber os pagamento o status irá passar de em "Aberto" para "Atrasado", já gerando uma notificação para os Gerentes da loja.
**RN06 —** Somente os usuários com o nível de Gerente ou Administrador podem cadastrar novos produtos, mudar manualmente o estatuas de uma compra ou puxar o histórico de vendas.

# 3. Requisitos Funcionais (mínimo: 8)
Liste os requisitos funcionais do seu MVP.

**RF01 —**  O Sistema irá mostrar o registro de venda que contem os produtos vendidos, informações do cliente e formas de pagamento utilizada.
**RF02 —**  O Sistema permite a pesquisa rápida por nome de produtos ou dados do cliente.
**RF03 —**  O Sistema sempre realiza uma verificação da disponibilidade de itens antes de efetuar uma venda.
**RF04 —**  O Sistema permite o controle de Contas a Receber com base em seus status (Abert, Recebida ou Atrasada)
**RF05 —**  O Sistema deve ser capaz de gerar relatórios dos produtos mais vendidos e menos vendidos com base no fluxo semanal e mensal.
**RF06 —**  O Sistema permite o controle total do estoque e modificações dos usuários com cargos de Desenvolvedor
**RF07 —**  O Sistema permitirá que usuários com cargo de gerente modificações manuais no estatuas da compra.
**RF08 —**  O Sistema permitirá que usuários com cargo de gerente registrar as Devoluções e perdas de produtos.

# 🛡 4. Requisitos Não Funcionais (mínimo: 4)
Liste os RNFs do sistema conforme seu MVP.

**RNF01 —**  Todos os dados devem ser criptografados, desde senhas, usuários além de ser armazenados de bancos de dados com os hash seguro.
**RNF02 —**  O Sistema sempre deve ter uma disponibilidade em horário comercial e realizando backup diários automáticos no momentos de menor movimento.
**RNF03 —**  Sempre que houver uma operação sensível será um registro de log com todos os dados de quem o realizou e qual foi a ação executada.
**RNF04 —**  A interface deve ser de fácil usabilidade e responsiva, fazendo que seja realizada no máximo 5 interações para terminar uma ação.

## **UC02  — Verificar Estoque**
**Ator(es):**  Sistema (chamado via <<include>> de UC01 e UC08)
**Descrição:**  Realização da Verificação do Estoque
**Pré-condições:**  Produto e unidade identificados.
**Pós-condições:**  Disponibilidade confirmada ou negada.

### Fluxo Principal
1. Sistema consulta saldo do produto na unidade.
2. Se quantidade ≥ solicitada: confirma disponibilidade.
3. Caso quantidade < solicitada: retorna indisponibilidade.
4. Quando quantidade ≤ nível mínimo: emite alerta ao gerente


### Fluxos Alternativos / Exceções
- FA01: Produto sem nível mínimo cadastrado → alerta não é emitido; operação prossegue.

### Relacionamentos
 <<include>> de UC01 e UC08
